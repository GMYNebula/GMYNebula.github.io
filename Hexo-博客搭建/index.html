  1. Hexo 博客搭建

    ​ Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

    ​ 我们可以轻松的将 Hexo 博客部署在 GitHub Pages 上，不需要再购买云服务器ESC。我通过官方文档以及一些其他资料列举了以下的几个步骤，借此方便在对 Hexo 博客有使用需求时的查询。

    一、准备工作

    1.1 安装

    • 管理：Git
    • 环境：Node.js（版本需高于10.13，建议使用12.x版本）
    • 应用： Hexo

    1.1.1 安装 Git

    1. 依照操作系统安装 Git:

      • Windows 版本下载地址：https://git-scm.com/download/win
      • Mac 版本下载安装程序：http://sourceforge.net/projects/git-osx-installer/
      • Linux (Ubuntu, Debian)指令：sudo apt-get install git-core
      • Linux (Fedora, Red Hat, CentOS)指令：sudo yum install git-core

    2. 使用cmd输入 git --version 检查Git版本：

    3. 完成（更详细的Git使用教程：Git 安装配置）

    1.1.2 安装 Node.js

    1. 通过官网下载安装 Node.js：https://nodejs.org/zh-cn/download/

      • 在 LTS (长期支持版 ) 中点击 Windows Installer (Windows 安装包 ) 下载 Windows 版本的安装包。

      • 在 LTS (长期支持版 ) 中点击 masOS Installer (masOS 安装包 ) 下载 masOS 版本的安装包。

      • Ubuntu指令：

    2. 使用cmd输入 node --version 检查Node.js版本：

    3. 完成（更详细的Git使用教程：Node.js 安装配置）

    1.1.3 安装 Hexo

    1. 打开刚刚安装的Git Bash

    2. 输入 npm install -g hexo-cli 安装

    3. 输入 hexo version 检查版本

    3. 新建网站所需要的文件：在 Git 中输入 hexo init 和 npm install

    9. 其他设置暂时不用修改，详情请见：Hexo 配置

    1.2.3 在本地创建网站

    1. 在Git Bash 中输入 hexo g 或 hexo generate 生成静态文件。

    2. 在Git Bash 中输入 hexo s 或 hexo server 启动服务器。默认情况下，访问网址为： http://localhost:4000/

      注：允许使用网络；保证运行这条命令时在网站项目文件夹路径下。

    3. 在浏览器中打开网址 http://localhost:4000/ ，就可以看到网站了。

    4. 在Git Bash 中使用快捷键 Ctrl + C 停止。

    这里仅用一个主题作为示范，可以到 Hexo 主题 浏览更多主题。

    2. 在Git Bash 中输入git clone https://github.com/geekrainy/hexo-theme-purely.git themes/purely

    3. 在**项目文件夹(./)**中修改 _config.yml 文件。找到 theme 修改为：

    4. 在**主题文件夹(./theme/purely)**中修改 _config.yml 文件，也可以不修改。详情可见：主题介绍

      1.2.5 发布文章

    5. 在Git Bash 中输入 hexo new "文章名" 在 ./source/_posts 创建一个 markdown文本文件。

    6. 使用markdown编辑器（例如：Typora）编写文件并保存。

    7. 刷新页面。文章是即时刷新的，所以不用重启服务器。

    1.2.6 为文章添加图片

    1. 确保网站配置中 post_asset_folder 为 true ，在使用 hexo new 时会创建一个与文章同名的文件夹。

    2. 在Git Bash 中输入 npm install hexo-asset-image --save 安装上传图片插件。

    3. 将图片存到这个与文章同名的文件夹。

    4. 在markdown文章中，图片使用相对路径。

1.3 将 Hexo 部署到 GitHub Pages

1.3.1 在 GitHub 创建 repository（仓库）

这个仓库用来存储生成好的静态网站

要保证 repository 的名字为你的 GitHub 的账号加.github.io

  1. 登录或注册 GitHub 账户。

  2. 创建仓库：

    1. 点击 GitHub 页面的右上角用户头像左边的加号中的 “New repository” ；

    2. 在 “Repository name” 中填入你的账号名加.github.io；

    3. 点击 “Create repository” 创建仓库。

    注：仓库创建不会立即完成需要等待一段时间（通常不会超过30分钟）。
